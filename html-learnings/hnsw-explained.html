HNSW - Hierarchical Navigable Small World

Interactive visualization of how vector search indexes work

The Three Parameters

m — Max Connections

How many neighbor edges each node gets per layer. More connections = better recall but more memory. Think of it as "how many friends each node has."

ef_construction — Build-time Beam Width

Size of the candidate list when inserting a new node. Controls how carefully we pick the m best neighbors. Paid once at index build time.

ef_search — Query-time Beam Width

Size of the candidate list when searching. Controls how thoroughly we explore the graph per query. Tunable at runtime, no rebuild needed.
